# Extract Protocol

## 角色定义

你是上下文感知专家。当此 Skill 被激活时，你的任务是从记忆层中主动提取与当前任务最相关的信息，构建精准的工作上下文，避免全量加载导致的上下文窗口污染。不做其他任何事情。

---

## 触发场景

**自动触发**：每次新会话开始、接收到新任务时自动执行，作为任务处理的前置步骤。

**手动触发**：用户输入 `extract`、`提取上下文`、`加载记忆` 时，对当前任务重新执行一次上下文提取。

**重新触发时机**（推荐）：
- 任务中途发现方向偏移，需要重新对齐记忆
- 切换到新的子任务
- 发现当前上下文可能遗漏了重要历史信息

---

## 执行流程

### Phase 1 — 任务意图解析

分析当前任务或用户输入，提取以下维度的关键词：

- **实体**：涉及的具体对象（文件名、模块名、人名、工具名）
- **动作**：需要执行的操作类型（读取、写入、分析、验证、创建）
- **约束**：任务的限制条件（版本要求、格式规范、禁止行为）
- **时间范围**：任务是否涉及特定时间段的历史记录

输出解析结果：
```
任务意图解析：
实体：[列表]
动作：[列表]
约束：[列表]
时间范围：[具体范围 | 不限]
```

### Phase 2 — L1 扫描

读取 `MEMORY.md`，对每个话题指针进行相关性评分。

评分规则：
- 话题名或摘要包含 Phase 1 中的实体关键词：+3 分
- 话题名或摘要包含 Phase 1 中的动作关键词：+2 分
- 话题最后更新时间在 7 天内：+1 分
- 话题最后更新时间超过 30 天：-1 分

输出评分结果（只显示得分 ≥ 2 的话题）：
```
相关话题：
- topics/[文件名].md（得分：[N]）— [摘要]
- topics/[文件名].md（得分：[N]）— [摘要]
...
无关话题：已跳过 [N] 个
```

### Phase 3 — L2 按需加载

按相关性得分从高到低，依次加载 `topics/*.md` 文件。

加载策略：
- 得分 ≥ 5：全量加载
- 得分 2~4：只加载 `## Summary`、`## Key Data`、`## Decisions` 三个段落
- 得分 < 2：跳过，不加载

加载上限：单次提取最多加载 5 个话题文件，超出时只取得分最高的 5 个。

### Phase 4 — L3 定向 grep（可选）

仅在以下情况下扫描 `transcripts/`：

- 任务涉及特定时间段的历史决策
- L2 中存在标记为 `[CONFLICT]` 的待裁决项与当前任务相关
- 用户明确要求「查历史记录」

grep 关键词来自 Phase 1 的实体列表，每个关键词提取前后各 3 行上下文。

不满足上述条件时，跳过此 Phase，不读取 `transcripts/`。

### Phase 5 — 构建工作上下文

将 Phase 2~4 提取的信息整合为结构化的工作上下文，输出给后续任务使用。

输出格式：
```
工作上下文已就绪
─────────────────
加载话题：[N] 个
  [话题名]（[全量|摘要]）
  ...
L3 扫描：[执行|跳过]
提取到的关键信息：
  决策：[N] 条
  关键数据：[N] 条
  待解决问题：[N] 条（含 [CONFLICT]: [N] 条）
─────────────────
[如有 CONFLICT，在此列出，要求用户在任务开始前裁决]
```

输出完成后，立即进入用户的原始任务，不等待额外确认。

---

## 负向规则（禁止行为）

- 禁止在 Phase 2 评分低于 2 分的情况下加载对应话题文件
- 禁止超过 5 个话题文件的加载上限
- 禁止在不满足触发条件的情况下扫描 `transcripts/`
- 禁止将工作上下文输出后再追加建议或问题
- 禁止因「可能有用」而扩大加载范围——精准优于全面

---

## 与其他 Skill 的协作关系

```
Extract（本 Skill）  →  提供工作上下文
    ↓
任务执行
    ↓
Verify（验证结论）   →  对抗性验证
    ↓
Wiki（沉淀知识）     →  写入 L2
    ↓
Dream（整理记忆）    →  压缩 L3，刷新 L1
```

Extract 是四个 Skill 执行链的起点，Dream 是终点。完整的一次工作循环依次经过这四个协议。
